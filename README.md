# Remote Debug Tunnel

VS Code에서 로컬로 실행 중인 개발 서버를 외부 기기에서 접근하여 디버깅할 수 있도록 하는 익스텐션입니다.

## 기능

- 🌐 로컬 포트를 인터넷에 공개하는 터널 생성
- 📱 QR 코드 생성으로 모바일 기기에서 쉽게 접근
- 📋 원클릭 URL 복사
- 🔔 상태 바에서 실시간 터널 상태 확인
- ⚙️ 사용자 정의 서브도메인 지원

## 사용 방법

### 1. 터널 시작하기

1. 명령 팔레트(`Cmd+Shift+P` 또는 `Ctrl+Shift+P`)를 엽니다
2. `Remote Debug: 터널 시작` 명령을 실행합니다
3. 로컬 포트 번호를 입력합니다 (예: 3000)
4. 원하는 서브도메인을 입력하거나 비워두면 랜덤으로 생성됩니다
5. 생성된 URL을 다른 기기에서 사용합니다!

### 2. 터널 상태 확인

- 상태 바의 `$(globe) 터널` 아이콘을 클릭
- 또는 명령 팔레트에서 `Remote Debug: 상태 확인` 실행

### 3. 터널 중지하기

- 명령 팔레트에서 `Remote Debug: 터널 중지` 실행

## 설정

### `remoteDebugTunnel.defaultPort`

- 타입: `number`
- 기본값: `3000`
- 설명: 기본 포트 번호

### `remoteDebugTunnel.autoStart`

- 타입: `boolean`
- 기본값: `false`
- 설명: 워크스페이스 열 때 자동으로 터널 시작

## 사용 예시

### React 앱 디버깅

```bash
# 터미널에서 React 앱 실행
npm start  # 기본적으로 포트 3000에서 실행됨

# VS Code에서:
# 1. Cmd+Shift+P
# 2. "Remote Debug: 터널 시작" 입력
# 3. 포트 3000 입력
# 4. 생성된 URL을 모바일 기기에서 열기
```

### Next.js 앱 디버깅

```bash
# 터미널에서 Next.js 앱 실행
npm run dev  # 기본적으로 포트 3000에서 실행됨

# VS Code에서 터널 시작하고 외부에서 접근
```

## 주의사항

- 터널은 공개 URL을 생성하므로 민감한 정보가 노출되지 않도록 주의하세요
- 개발 목적으로만 사용하세요
- 터널 사용이 끝나면 반드시 중지하세요

## 문제 해결

### 터널 생성 실패

- 입력한 포트에서 서버가 실행 중인지 확인하세요
- 방화벽 설정을 확인하세요
- 다른 포트로 시도해보세요

### 연결 불가

- 로컬 서버가 실행 중인지 확인하세요
- 터널 URL이 올바른지 확인하세요
- 인터넷 연결을 확인하세요
